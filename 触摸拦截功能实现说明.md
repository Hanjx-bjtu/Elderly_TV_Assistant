# 触摸拦截功能实现说明

## 问题描述

由于应用使用电脑版UserAgent加载央视网页面，导致网页显示桌面版界面，包含可点击元素（播放控制、广告、链接等）。当用户点击屏幕左右区域进行换台时，会同时触发：
- ✅ 应用的换台功能
- ❌ 网页的点击事件（可能跳转、暂停播放等）

这严重影响了用户体验，特别是对于老年用户。

## 解决方案

采用**优化方案：在GestureRelativeLayout中拦截触摸事件**，通过父布局直接拦截触摸事件，防止WebView接收触摸事件，同时保持底部控制栏按钮正常工作。

## 实现细节

### 1. 修改GestureRelativeLayout拦截触摸事件

**文件**: [GestureRelativeLayout.kt](file:///d:\Programming&Labs\Mobile_application_development\DesignWork\app\src\main\java\com\elderly\tvassistant\widget\GestureRelativeLayout.kt)

```kotlin
class GestureRelativeLayout @JvmOverloads constructor(
    context: Context,
    attrs: AttributeSet? = null,
    defStyleAttr: Int = 0
) : RelativeLayout(context, attrs, defStyleAttr) {

    companion object {
        private const val CLICK_THRESHOLD = 50  // 点击位移阈值（像素）
        private const val BOTTOM_BAR_HEIGHT_THRESHOLD = 250  // 底部控制栏高度阈值（像素）
    }

    /**
     * 拦截触摸事件，防止传递给WebView
     * 但不拦截底部控制栏区域的触摸事件，让按钮正常工作
     */
    override fun onInterceptTouchEvent(event: MotionEvent): Boolean {
        when (event.action) {
            MotionEvent.ACTION_DOWN -> {
                touchStartX = event.x
                touchStartY = event.y
                
                // 检查触摸点是否在底部控制栏区域
                val isInBottomBar = event.y > (height - BOTTOM_BAR_HEIGHT_THRESHOLD)
                
                // 如果在底部控制栏区域，不拦截，让按钮正常工作
                // 否则拦截，防止WebView接收触摸事件
                return !isInBottomBar
            }
        }
        return false
    }

    override fun onTouchEvent(event: MotionEvent): Boolean {
        when (event.action) {
            MotionEvent.ACTION_DOWN -> {
                touchStartX = event.x
                touchStartY = event.y
                return true
            }
            MotionEvent.ACTION_UP -> {
                val deltaX = Math.abs(event.x - touchStartX)
                val deltaY = Math.abs(event.y - touchStartY)
                // 判定为点击（非滑动）
                if (deltaX < CLICK_THRESHOLD && deltaY < CLICK_THRESHOLD) {
                    val x = event.x
                    val width = width.toFloat()
                    when {
                        x < width / 3 -> listener?.onLeftClick()
                        x > width * 2 / 3 -> listener?.onRightClick()
                        else -> listener?.onMiddleClick()
                    }
                }
                return true
            }
        }
        return super.onTouchEvent(event)
    }
}
```

**关键点**:
- `onInterceptTouchEvent`：根据触摸位置决定是否拦截
- 视频区域：拦截触摸事件，防止WebView接收
- 底部控制栏区域：不拦截，让按钮正常工作
- `onTouchEvent`：处理手势，触发换台或显示/隐藏控制栏

### 3. 禁用WebView触摸事件

**文件**: [PlayerManager.kt](file:///d:\Programming&Labs\Mobile_application_development\DesignWork\app\src\main\java\com\elderly\tvassistant\manager\PlayerManager.kt)

```kotlin
private fun initWebView() {
    webView = WebView(context).apply {
        // ... 其他配置 ...

        // 禁用WebView的触摸事件处理，所有触摸由上层GestureRelativeLayout拦截
        isClickable = false
        isFocusable = false
        isFocusableInTouchMode = false
        setOnTouchListener(null)

        // ... 其他配置 ...
    }
}
```

**关键点**:
- `isClickable = false`：禁用点击
- `isFocusable = false`：禁用焦点
- `isFocusableInTouchMode = false`：禁用触摸模式焦点
- `setOnTouchListener(null)`：移除触摸监听器

## 工作原理

### 触摸事件流程

#### 视频区域触摸
```
用户触摸视频区域
    ↓
GestureRelativeLayout.onInterceptTouchEvent()
    ↓ (判断不在底部控制栏区域，返回true)
拦截触摸事件，不传递给WebView
    ↓
GestureRelativeLayout.onTouchEvent()
    ↓ (处理手势)
触发换台功能
```

#### 底部控制栏区域触摸
```
用户触摸底部控制栏区域
    ↓
GestureRelativeLayout.onInterceptTouchEvent()
    ↓ (判断在底部控制栏区域，返回false)
不拦截触摸事件，传递给子View
    ↓
LinearLayout/按钮接收触摸事件
    ↓
按钮功能正常工作
```

### 事件传递说明

1. **GestureRelativeLayout拦截阶段**:
   - 检查触摸点位置
   - 如果在视频区域：返回 `true`，拦截触摸事件
   - 如果在底部控制栏区域：返回 `false`，不拦截触摸事件

2. **视频区域处理**:
   - 拦截的触摸事件由 `onTouchEvent` 处理
   - 根据触摸位置判断是左、中、右哪个区域
   - 触发相应的回调（换台、显示/隐藏控制栏）

3. **底部控制栏区域处理**:
   - 触摸事件正常传递给子View
   - 按钮可以正常接收点击事件
   - 按钮功能不受影响

## 优势

### ✅ 优点

1. **完全透明**: 不需要额外的透明View，视频完全可见
2. **完全控制触摸事件**: WebView无法接收任何触摸事件，彻底避免网页干扰
3. **保持现有交互**: 不改变现有的左右点击换台逻辑
4. **按钮正常工作**: 底部控制栏的触摸事件不受影响
5. **适老化设计**: 符合老年人简单直观的操作习惯
6. **性能优化**: 无额外View，不影响性能
7. **代码简洁**: 不需要额外的拦截层View

### ⚠️ 注意事项

1. **无法使用网页控制**: 由于WebView触摸被禁用，用户无法使用网页内的播放控制
   - **解决方案**: 应用已有完整的控制栏（播放、暂停、换台等）

2. **底部控制栏高度**: 需要根据实际布局调整 `BOTTOM_BAR_HEIGHT_THRESHOLD` 值
   - **当前设置**: 250像素，可根据实际情况调整

3. **全屏视频**: 如果网页视频支持全屏，可能需要额外处理
   - **当前状态**: 央视网页面已通过JavaScript隐藏其他元素，只保留播放器

## 总结

通过在GestureRelativeLayout中拦截触摸事件，成功解决了WebView网页点击干扰应用手势操作的问题。该方案：
- ✅ 完全拦截WebView触摸事件
- ✅ 保持应用手势功能正常
- ✅ 不影响底部控制栏按钮
- ✅ 视频完全可见，无遮挡
- ✅ 适合老年用户使用
- ✅ 代码简洁，性能优异

这是一个简单、有效、可靠的解决方案，完全符合适老化设计的要求。
