## 1. 引言

### 1.1 编写目的

本文档为《长辈电视助手》Android应用的需求规格说明书，旨在：
- 明确系统的功能需求与非功能需求
- 作为开发、测试和验收的依据
- 作为课程设计报告的重要组成部分

### 1.2 项目背景

随着智能手机的普及，越来越多的老年人开始使用智能手机。然而，主流APP的界面设计复杂、字体偏小、操作逻辑繁琐，对老年用户极不友好。本项目旨在开发一款**专为老年人设计的电视直播观看助手**，通过大字版界面、语音交互、简化手势等适老化设计，帮助老年人轻松观看电视节目。

### 1.3 目标用户

| 用户类型 | 描述 | 典型特征 |
|---------|------|----------|
| 主要用户 | 65岁以上老年人 | 视力下降、手指灵活性降低、对智能设备操作不熟练 |
| 次要用户 | 视力不佳或手部精细操作能力下降的人群 | 需要大字体、简化操作 |
| 协助者 | 老人的子女或照护者 | 负责初始安装、设置和简单指导 |

### 1.4 开发环境

| 项目 | 规格 |
|------|------|
| 操作系统 | Windows 11 |
| 开发工具 | Android Studio Panda |
| 编程语言 | Java / Kotlin（混合使用），主要使用Kotlin |
| 最低SDK版本 | API 26 (Android 8.0) |
| 目标SDK版本 | API 34 (Android 14) |
| 测试设备 | 真机 + 模拟器 |

## 2. 系统概述

### 2.1 系统定位

《长辈电视助手》是一款**专为老年用户设计的移动端电视直播观看应用**，核心价值在于：
- **极简操作**：进入即播放，左右点击换台
- **适老设计**：大字体、高对比度、大按钮
- **多模态交互**：触摸 + 语音双重支持

### 2.2 系统范围

本项目涵盖以下范围：
- Android客户端开发（不包含后端服务器）
- 本地数据存储（频道列表、收藏、设置）
- 第三方服务集成（语音识别、语音合成）

本项目**不包含**以下内容：
- 视频源服务器搭建
- 用户账号系统
- 后台管理界面

### 2.3 约束条件

| 约束类型 | 描述 |
|---------|------|
| 时间约束 | 开发周期约2-3周（课程设计期限） |
| 人力约束 | 开发人员2人 |
| 技术约束 | 必须使用Android原生开发，不得使用跨平台框架 |
| 资源约束 | 视频源依赖第三方（央视网），不得自建流媒体服务器 |

### 2.4 假设与依赖

| 类型 | 内容 |
|------|------|
| 假设 | 用户手机已连接网络 |
| 假设 | 用户授予必要的权限（网络、录音） |
| 依赖 | 央视网移动版页面稳定可用 |
| 依赖 | Android设备支持语音识别服务 |

### 3.2 功能详细描述

#### 3.2.1 核心播放功能

| 功能ID | F-01 |
|--------|------|
| **功能名称** | 自动播放 |
| **优先级** | 高 |
| **描述** | 用户启动APP后，自动开始播放默认频道（默认为CCTV-1） |
| **前置条件** | 网络连接正常、权限已授予 |
| **后置条件** | 视频正常播放 |
| **异常处理** | 网络异常时显示提示，播放失败时显示错误页面 |
| **验收标准** | 启动APP → 自动播放CCTV-1 |

| 功能ID | F-02 |
|--------|------|
| **功能名称** | 左右点击切换频道 |
| **优先级** | 高 |
| **描述** | 用户点击屏幕左侧区域切换到上一个频道，点击右侧区域切换到下一个频道 |
| **交互方式** | 触摸点击（非滑动） |
| **反馈** | 切换时有震动（可选）+ 语音播报频道名 |
| **验收标准** | 点击左侧 → 切换到上一频道；点击右侧 → 切换到下一频道 |

| 功能ID | F-03 |
|--------|------|
| **功能名称** | 频道循环 |
| **优先级** | 中 |
| **描述** | 到达频道列表末尾后，继续切换回到第一个频道 |
| **验收标准** | 最后一个频道点下一频道 → 切换到第一个频道 |

#### 3.2.2 频道管理功能

| 功能ID | F-04 |
|--------|------|
| **功能名称** | 频道列表展示 |
| **优先级** | 高 |
| **描述** | 侧边栏展示所有可用频道，每个频道项显示频道名称和图标 |
| **UI要求** | 字体大（最小20sp），行高至少72dp |
| **交互** | 点击任意频道项直接切换至该频道 |
| **验收标准** | 打开侧边栏 → 显示频道列表 → 点击频道 → 切换成功 |

| 功能ID | F-05 |
|--------|------|
| **功能名称** | 频道收藏 |
| **优先级** | 中 |
| **描述** | 用户可将常看频道加入收藏列表，一键快速访问 |
| **操作方式** | 频道列表项右侧显示"收藏"星标，点击切换收藏状态 |
| **数据存储** | 收藏数据本地持久化（Room数据库） |
| **验收标准** | 收藏频道 → 在"我的收藏"中显示 → 可快速切换 |

| 功能ID | F-06 |
|--------|------|
| **功能名称** | 当前频道显示 |
| **优先级** | 高 |
| **描述** | 屏幕底部（或顶部）大字显示当前正在播放的频道名称 |
| **UI要求** | 字体不小于32sp，高对比度颜色 |
| **验收标准** | 切换频道后 → 频道名称同步更新 |

#### 3.2.3 语音交互功能

| 功能ID | F-07 |
|--------|------|
| **功能名称** | 语音切换频道 |
| **优先级** | 高 |
| **描述** | 用户点击语音按钮（或长按屏幕任意区域）后说出频道名称，系统自动切换到对应频道 |
| **触发方式** | 点击底部语音按钮 |
| **识别范围** | 预设频道关键词（如“中央一台”、“央视一套”、“综合频道”等） |
| **反馈** | 识别中显示动画，识别成功/失败有提示 |
| **验收标准** | 点击语音按钮 → 说“中央一台” → 切换到CCTV-1 |

| 功能ID | F-08 |
|--------|------|
| **功能名称** | 频道语音播报 |
| **优先级** | 中 |

| **开关控制** | 支持在设置中关闭 |
| **验收标准** | 切换频道 → 听到“CCTV-1 综合”语音播报 |

#### 3.2.4 定时关闭功能

| 功能ID | F-09 |

| **取消方式** | 在定时未到期前可取消 |
| **通知提醒** | 定时结束时发送通知“播放时间已到” |
| **验收标准** | 设置30分钟定时 → 30分钟后APP退出并收到通知 |

## 4. 非功能需求

### 4.1 性能需求

| 需求ID | 需求描述 | 指标 |
|--------|---------|------|
| N-01 | APP启动到播放的响应时间 | ≤ 3秒 |
| N-02 | 频道切换响应时间 | ≤ 1秒 |
| N-03 | 语音识别延迟 | ≤ 2秒 |
| N-04 | APP内存占用 | ≤ 150MB |
| N-05 | 后台耗电 | 无定时任务时几乎不耗电 |

### 4.2 可用性需求

| 需求ID | 需求描述 |
|--------|---------|
| N-06 | 所有可点击元素尺寸不小于72×72dp |
| N-07 | 文本最小字号不小于16sp（普通模式）/26sp（超大模式） |
| N-08 | 主要操作步骤不超过2步（从主页到目标功能） |
| N-09 | 错误提示使用口语化语言，不显示技术性错误码 |
| N-10 | 所有图标配有文字标签 |

### 4.3 兼容性需求

| 需求ID | 需求描述 |
|--------|---------|
| N-11 | 支持Android 8.0（API 26）及以上版本 |
| N-12 | 支持常见屏幕尺寸（5.0寸 ~ 6.8寸） |
| N-13 | 支持常见分辨率（720p、1080p、2K） |
| N-14 | 适配主流品牌（华为、小米、OPPO、vivo、三星） |

### 4.4 安全性需求

| 需求ID | 需求描述 |
|--------|---------|
| N-15 | 语音数据不上传服务器，仅本地处理（Android SpeechRecognizer规范） |
| N-16 | 不收集用户个人信息 |
| N-17 | 网络请求仅用于视频播放和频道数据加载 |

### 4.5 可维护性需求

| 需求ID | 需求描述 |
|--------|---------|
| N-18 | 代码遵循MVVM架构模式 |
| N-19 | 关键模块有注释，注释率不低于20% |
| N-20 | 频道数据通过JSON配置文件管理，便于后续更新 |

## 5. 数据需求

### 5.1 本地数据存储

| 数据实体 | 存储方式 | 数据量预估 |
|---------|---------|-----------|
| 频道列表 | Room数据库 | ≤ 50条 |
| 收藏列表 | Room数据库 | ≤ 30条 |
| 用户设置 | SharedPreferences | ≤ 10项 |

### 5.2 数据实体定义

**实体1：Channel（频道）**

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | Integer | 是 | 主键，自增 |
| name | String | 是 | 频道显示名称 |
| keywords | List\<String\> | 是 | 语音匹配关键词 |
| url | String | 是 | 视频源地址 |
| iconResId | Integer | 否 | 图标资源ID |
| sortOrder | Integer | 是 | 排序序号 |

**实体2：Favorite（收藏）**

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | Integer | 是 | 主键，自增 |
| channelId | Integer | 是 | 关联Channel.id |
| createdAt | Long | 是 | 收藏时间戳 |

**实体3：Setting（设置）**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| fontSizeLevel | Integer | 3（超大） | 1:普通 2:大 3:超大 |
| ttsEnabled | Boolean | true | 语音播报开关 |
| vibrateEnabled | Boolean | true | 震动反馈开关 |
| defaultChannelId | Integer | 1 | 默认频道ID |
| firstLaunchDone | Boolean | false | 是否已完成首次引导 |

## 8. 项目约束与风险

### 8.1 技术约束

- 必须使用Android原生开发（Java/Kotlin）
- 不得使用第三方跨平台框架（如Flutter、React Native）
- 必须包含课程要求的核心组件（至少5类）

### 8.2 时间约束

| 阶段 | 时间 | 交付物 |
|------|------|--------|
| 需求分析与设计 | 第1-2天 | 需求规格说明书、详细设计说明书 |
| 核心功能开发 | 第3-8天 | 可运行Demo |
| 功能完善与测试 | 第9-12天 | 完整APK |
| 文档与报告 | 第13-14天 | 课程设计报告、演示视频 |

### 8.3 风险识别与应对

| 风险ID | 风险描述 | 可能性 | 影响程度 | 应对措施 |
|--------|---------|--------|---------|---------|

| R-04 | 两人开发进度协调问题 | 中 | 中 | 每日站会，使用Git进行版本管理 |
| R-05 | 定时关闭在部分厂商ROM被限制 | 中 | 低 | 增加前台服务通知，引导用户手动设置 |
