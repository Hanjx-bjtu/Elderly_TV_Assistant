# 《长辈电视助手》软件设计草案（第二版）

## 一、设计目的

为家中不熟悉智能手机复杂操作的老年人，设计一款**大字版、交互简单、语音友好**的电视直播观看助手。通过极简的操作逻辑和针对老年人使用习惯的交互优化，让老人能够轻松观看CCTV等电视节目，降低使用门槛，提升生活便利性与幸福感。

## 二、目标用户

- **主要用户**：65岁以上、使用智能手机但不熟练的老年人
- **次要用户**：视力不佳、手部精细操作能力下降的人群
- **协助者**：老人的子女或照护者（协助安装与初始设置）

## 三、核心设计原则（适老化）

| 原则 | 说明 | 技术体现 |
|------|------|----------|
| **可见即可点** | 所有可操作元素足够大、颜色鲜明 | 使用 `sp` 单位、高对比度主题 |
| **操作容错** | 减少误触影响，关键操作需确认 | 定时关闭增加确认对话框 |
| **多模态交互** | 支持触摸 + 语音，适应不同能力老人 | 语音换台 + 语音反馈（TTS） |
| **简化手势** | 只使用点击、左右点击，避免滑动/长按 | 手势检测只识别点击 |
| **视觉一致性** | 统一的字体、色彩、间距，降低认知负担 | 自定义Theme、颜色资源文件 |

## 四、功能模块设计

### 4.1 核心功能（MVP，必做）

| 功能 | 描述 | 交互方式 |
|------|------|----------|
| **视频播放** | 播放CCTV频道（默认CCTV1） | 进入APP即开始播放 |
| **频道切换（左/右）** | 切换到上一个/下一个频道 | 点击屏幕左侧区域/右侧区域 |
| **频道列表** | 展示所有可用频道 | 侧边抽屉菜单（大字版） |
| **当前频道显示** | 屏幕顶部显示当前频道名称（大字） | 文字+图标 |
| **退出提示** | 按返回键时提示是否退出 | 对话框确认 |

### 4.2 扩展功能（选做，用于技术覆盖面拉分）

| 优先级 | 功能 | 描述 | 技术实现 |
|--------|------|------|----------|
| ⭐⭐⭐ | **语音换台** | 老人说出“中央一台”、“换到体育频道”等指令切换频道 | `SpeechRecognizer` + 自然语言匹配 |
| ⭐⭐⭐ | **定时关闭** | 设置播放时长（30/60/90分钟），到时自动停止播放并退出 | `AlarmManager` + `Notification` |
| ⭐⭐⭐ | **频道语音播报** | 切换频道时，自动用语音读出当前频道名称 | `TextToSpeech` |
| ⭐⭐ | **频道收藏** | 老人可将常看频道加入“最爱列表”，一键切换 | `Room` + `RecyclerView` |
| ⭐⭐ | **开机引导/说明** | 首次启动时展示简单的使用说明（图片+大文字） | `SharedPreferences` + `ViewPager2` |
| ⭐ | **桌面小部件** | 在手机桌面放置一键打开APP的小部件 | `AppWidgetProvider` |
| ⭐ | **震动反馈** | 点击按钮时提供轻微震动，确认操作 | `Vibrator` |

### 4.3 频道数据来源（技术选型关键）

| 方案 | 实现 | 优点 | 缺点 | 推荐度 |
|------|------|------|------|--------|
| 央视网移动版（H5） | `WebView` 加载 `https://m.cctv.com` | 稳定、合法、无需维护 | 可能有广告，非纯视频 | ⭐⭐⭐⭐⭐ |
| 公开 m3u8 直播源 | `IjkPlayer` / `ExoPlayer` 播放 | 纯视频体验好 | 源可能失效，仅用于课程演示 | ⭐⭐⭐ |
| 后端代理爬取 | 自建简易后端 | 可控性强 | 工作量大，超出课程范围 | ⭐ |

**推荐方案**：**主方案用 WebView 加载央视网移动版**（稳定可靠），**备选/演示方案用 1~2 个公开 m3u8 源**展示纯视频播放能力。

## 五、技术组件覆盖（对标课程评分标准）

| 类别 | 技术点 | 对应功能 |
|------|--------|----------|
| **UI组件** | `Fragment`、`RecyclerView`、`DrawerLayout`、`CardView`、`ViewPager2` | 频道列表、收藏页、引导页、侧边菜单 |
| **数据存储** | `Room`、`SharedPreferences` | 收藏频道、用户设置、引导状态 |
| **网络** | `WebView` / `OkHttp` | 视频播放 / 获取频道列表JSON |
| **后台任务** | `AlarmManager`、`Service`（可选） | 定时关闭 |
| **系统服务** | `SpeechRecognizer`、`TextToSpeech`、`Vibrator`、`NotificationManager` | 语音换台、语音播报、震动反馈、通知提醒 |
| **多媒体** | `MediaPlayer` / `IjkPlayer` / `WebView` | 视频播放 |
| **第三方库** | `Glide`（频道图标）、`MPAndroidChart`（可选使用统计） | UI增强、数据可视化 |

## 六、页面结构与交互流程

```
启动页（Splash）
    ↓
主界面（MainActivity）
├── 顶部状态栏（沉浸式）
├── 视频播放区域（占70%屏幕）
├── 当前频道名称（大字，居中）
├── 左侧点击区域（切换上一频道）
├── 右侧点击区域（切换下一频道）
├── 底部辅助按钮（收藏、定时、语音）
└── 侧边抽屉菜单（频道列表 + 最爱）
```

### 核心交互流程

1. **启动APP** → 默认播放 CCTV-1
2. **切换频道** → 点击屏幕左侧/右侧 → 更新播放源 + 语音播报频道名
3. **语音换台** → 点击语音按钮（或长按屏幕）→ 说出频道名 → 自动切换
4. **打开频道列表** → 从左侧向右滑动（或点击菜单图标）→ 点击列表项切换
5. **定时关闭** → 点击定时按钮 → 选择时长 → 启动倒计时 → 到时通知并退出

## 七、数据设计（简化版）

### 7.1 频道数据结构（`Channel`）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | int | 主键 |
| `name` | String | 频道名称（如“CCTV-1 综合”） |
| `keyword` | String | 语音匹配关键词（如“中央一台”、“央视一套”） |
| `url` | String | 视频源地址（m3u8或网页地址） |
| `icon` | String | 频道图标URL（可选） |
| `isFav` | boolean | 是否收藏 |

### 7.2 本地存储

- **Room数据库**：存储频道列表 + 收藏状态
- **SharedPreferences**：存储定时时长、引导完成标志、音量设置等

### 7.3 频道数据初始化

首次启动时，从 `assets/channels.json` 读取预设频道列表，写入Room数据库。

示例预设频道：
```json
[
  {"name": "CCTV-1 综合", "keyword": ["中央一台","央视一套","综合频道"], "url": "https://..."},
  {"name": "CCTV-2 财经", "keyword": ["中央二台","财经频道"], "url": "https://..."},
  {"name": "CCTV-3 综艺", "keyword": ["中央三台","综艺频道"], "url": "https://..."},
  {"name": "CCTV-4 中文国际", "keyword": ["中央四台","中文国际"], "url": "https://..."},
  {"name": "CCTV-5 体育", "keyword": ["中央五台","体育频道"], "url": "https://..."}
]
```

## 九、预期难点与解决思路

| 难点 | 解决思路 |
|------|----------|
| **视频源稳定性** | 主用WebView + 央视官网，备选m3u8源仅用于演示；报告中注明仅用于课程学习 |
| **语音识别准确率** | 限制匹配的词库（仅预设频道关键词），不开放自由语义；增加TTS回读确认 |
| **不同分辨率适配** | 使用 `ConstraintLayout` + 相对布局；字体用 `sp`，按钮用 `dp` 确保比例 |
| **定时关闭与播放器冲突** | 定时到点后发送广播，Activity接收后调用 `finishAffinity()` 或 `moveTaskToBack()` |
| **老人误触返回键** | 重写 `onBackPressed()`，弹出确认对话框“是否退出？” |
